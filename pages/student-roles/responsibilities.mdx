# Defining Responsibilities

Each role needs a clear job description that students can reference throughout the project.

## Role Description Template

For each role, define:

- **Title** — The role name
- **Purpose** — Why this role exists on the team
- **Key responsibilities** — 3-5 specific tasks this person owns
- **Deliverables** — What this person produces
- **Skills developed** — Which learning goals this role addresses

## Sample Role Description

**Title:** Data Analyst

**Purpose:** Responsible for collecting, organizing, and interpreting quantitative data to support the team's conclusions.

**Key Responsibilities:**
1. Design data collection instruments (surveys, observation protocols)
2. Organize raw data into usable formats
3. Create data visualizations (charts, graphs, tables)
4. Write the data analysis section of the final report
5. Present findings to the team for discussion

**Deliverables:**
- Completed data collection spreadsheet
- 3+ data visualizations
- Written analysis summary (500-750 words)

**Skills Developed:** Tier 2 (data analysis), Tier 3 (interpretation and evaluation), Tier 4 (visual communication)

## AI Prompt for Role Generation

```
Design 4 student roles for a PBL mission with this context:
[describe mission context]

For each role, provide:
- Title (use a professional title, not "Student 1")
- Purpose (one sentence)
- 3-5 key responsibilities
- Required deliverables
- Which learning goal tiers the role develops

Roles should have minimal overlap and collectively cover all project tasks.
```
